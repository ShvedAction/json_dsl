# Brain dump: boolean expressions + find

## Задача

Нужны операторы, которые строят условные булиновские выражения, и **`find`** по коллекции.

Целевой паттерн (JS):

```javascript
collection.find(itm =>
  itm.nested_collection.find(itm_nested =>
    itm_nested.some_property == 'some_string_value'
  )
).prop.path;
```

## Семантика

- Сравнение значений (минимум равенство `==`)
- Логические комбинаторы для построения предикатов
- `find` — первый элемент коллекции, для которого предикат истинен
- Вложенный `find` в предикате (как в примере)
- После нахождения — чтение поля по path (`prop.path`)

## Открытые вопросы

- Полный набор операторов: `eq` only vs `and`/`or`/`not`
- Поведение если `find` ничего не нашёл: null vs throw
- Сравнение только строк или любых JSON-значений
